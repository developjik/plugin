# QA Good Example

## Context
Running app was exercised through real user flows.

## What Good Looks Like
- Bugs include reproduction steps and actual evidence.
- Pass verdict is justified by tested flows.
- Missing edge cases are called out explicitly.

## Expected Evaluation
- verdict: pass only with concrete evidence
- findings: specific and reproducible

## Why This Matters
- Use this when the evaluator starts hand-waving instead of testing.
