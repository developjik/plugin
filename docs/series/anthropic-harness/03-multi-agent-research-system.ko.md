# How we built our multi-agent research system

- 원문: [How we built our multi-agent research system](https://www.anthropic.com/engineering/multi-agent-research-system)
- 출처: Anthropic Engineering
- 게시일: 2025-06-13

## 한줄 요약

복잡한 리서치 문제에서는 한 에이전트가 모든 것을 기억하려 하기보다, 리드 에이전트가 서브에이전트를 병렬로 돌려 요약만 모으는 방식이 더 강력하다.

## 이 글이 답하는 질문

멀티에이전트가 실제로 단일 에이전트보다 나아지는 문제는 무엇인가?

## 핵심 주장

- 리서치는 병렬 탐색과 증거 수집이 중요하므로 orchestrator-worker 구조와 잘 맞는다.
- 서브에이전트는 깊게 탐색하고, 리드 에이전트는 요약만 받아 합성한다.
- 각 서브에이전트가 독립 컨텍스트를 쓰면 전체 컨텍스트 오염이 줄어든다.

## 하네스 관점의 의미

멀티에이전트의 진짜 장점은 "여러 명이 동시에 일한다"보다 "탐색용 컨텍스트를 분리 저장한다"에 있다. 세부 탐색은 서브에이전트 안에서 소모하고, 리드 에이전트에는 압축된 결과만 남긴다.

## 실무 포인트

- 병렬화 이득이 큰 문제에만 쓸 것
- 강하게 결합된 구현 작업에는 오히려 조정 비용이 커질 수 있음
- 최종 답변 전 citation/verification 단계를 따로 두는 것이 좋음

## 기억할 문장

멀티에이전트의 핵심은 분업 자체보다, 탐색 맥락을 격리하고 요약만 끌어올리는 데 있다.
